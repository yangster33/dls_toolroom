# 法

高程查询遵循标准化的数据获取流程，确保查询结果的准确性和可靠性。

## 查询流程

1. **输入验证**：首先对用户输入的经纬度进行格式检查和范围验证，确保坐标值在有效范围内（纬度-90至90，经度-180至180）。

2. **API请求构建**：将验证通过的坐标参数按照 Open-Meteo API 的规范格式进行封装，构建符合要求的请求URL。

3. **数据获取**：通过 HTTP 请求向 Open-Meteo Elevation API 发起查询，获取高程数据。

4. **结果解析**：对 API 返回的 JSON 数据进行解析，提取高程值，并将其与原始坐标关联。

5. **结果展示**：以清晰直观的方式展示查询结果，包括原始坐标和对应的高程值。

## 设计原则

- **准确性**：确保查询结果准确反映实际海拔高度
- **可靠性**：对异常情况进行妥善处理，提供友好的错误提示
- **易用性**：简化操作流程，让用户快速获取所需信息
