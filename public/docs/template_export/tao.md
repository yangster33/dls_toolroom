# 道

信息的生产与信息的呈现，是同一枚硬币的两面。结构化数据承载着事实与记录，而文档则是这些事实面向人类阅读者的最终形态。当数据量从个位数增长到成百上千条时，逐条手工将数据填入文档就不再是 labor（劳动）而是 toil（苦役）——重复、低效、易错，且完全无法规模化。

批量文档生成所要解决的本质命题，是在数据与呈现之间建立一条可复用的转换通道。这条通道的核心思想是"分离关注点"：数据的维护者专注于数据的准确性与完整性，文档的设计者专注于版式、排版与视觉表达，而二者之间的映射关系被抽象为一套占位符号约定。一旦这套约定确立，任何符合格式的数据都可以自动流入预先设计好的文档骨架，生成最终成品。这不仅是效率的提升，更是对人类注意力资源的尊重——让人去做判断与创造，让机器去做搬运与填充。
